Skipping gradient/Hessian convergence checks

We can save time (not much in this case) by skipping the convergence tests based on the gradient and Hessian (at the expense of missing this information).

(t2 <- system.time(update(mp1,
             control=glmerControl(calc.derivs=FALSE))))

##    user  system elapsed 
##  49.515   0.364  49.919

(as of the current version, update() doesn't use any tricks that would make it faster than re-running the model from scratch– e.g. starting from last achieved parameter values).

The number of function evaluations needed for the gradient/Hessian scales as \(p(p+1)/2\); in this case the model has 27 parameters (378 evaluations for the Hessian) parameters, but the entire fit takes 5848 evaluations, so that doesn't actually save us very much …

If we use nloptwrap and shut off the Hessian checking, we can speed up quite a bit:

(t3 <- system.time(mp1B <- update(mp1,
             control=glmerControl(optimizer=nloptwrap,
                         calc.derivs=FALSE))))

##    user  system elapsed 
##   12.60    0.04   12.65

The fit gets within 0.0074 log-likelihood units of the original fit, and is obviously a lot faster (it only uses 113 function evaluations!)
